# Mitata 交互展示 - 开发指南

## 项目架构

### 核心模块

1. **人脸检测模块**
   - 使用 MediaPipe Face Mesh
   - 468 个面部关键点
   - 启用虹膜检测（refineLandmarks: true）

2. **平滑处理模块**
   - 移动平均算法
   - 8 帧缓冲区
   - 对所有关键点进行平滑

3. **渲染模块**
   - 双 Canvas 架构
   - 下半部分：完整视频 + 文本效果
   - 上半部分：放大的纯文本效果

## 开发建议

### 性能优化

1. **减少 Canvas 重绘**
   - 只在检测到人脸时更新
   - 使用 requestAnimationFrame

2. **平滑算法调优**
   - 增加缓冲区大小可以更平滑，但会增加延迟
   - 当前设置：8 帧缓冲

3. **文本渲染优化**
   - 按字符渲染而非整词
   - 使用裁剪路径减少绘制范围

### 常见问题

**Q: 文本抖动明显？**
A: 增加 `smoothingBuffer.maxSize` 的值（当前为 8）

**Q: 瞳孔检测不准确？**
A: 确保 `refineLandmarks: true` 已启用，并且光线充足

**Q: 性能问题？**
A: 降低视频分辨率或减少文本密度
